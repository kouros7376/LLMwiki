## 한 줄 요약
PMS/PMIS는 **건설·설계 생애주기 전반(계획·실행·관리)을 시스템에 일원화**하는 사업정보 관리 시스템. 발주처/CM/CS/시공사/**설계사** 별로 적용 형태가 다름. 핵심 가치: 단위 사업장별 손익 가시화 + 회계 일원화 + 본사 업무 효율화.

## 핵심 개념

- **PMS (Project Management System)** — 프로젝트 단위 일정/비용/품질 관리
- **PMIS (Project Management Information System)** — 사업 정보 관리에 더 광범위. 문서·도면·보고·승인 워크플로 포함
- 설계사무소에서 PMS는 보통: 수주 → 계약 → 설계 단계별 진행률 → 외주 → 인허가 → 준공 → 정산 → A/S 의 흐름을 다룸

## 대림산업 사례 (대표 케이스)

**1단계 (1998~2000)**: 건설 ERP 구축
- 인사·영업·외주·회계·자금·자재 등 내부관리 + 공사관리 프로세스 통합
- 본사 중앙집중 관리 체제 구축

**2단계: 손익관리 일원화**
- 회계상 손익과 공사 손익의 차이를 일원화
- 준공 시 예정 원가 정확 산출
- 경영관리 일관성 확보

**도출된 효과**
1. 단위 사업장별 손익관리 용이
2. 프로세스 중심 업무 처리 → 본사 업무 효율화
3. 실시간 처리 → 업무 리드타임 단축
4. 동일 데이터 중복관리 배제 → 자료 일관성
5. 의사결정 정보 신속 확보

> **시사점 (설계사무소 적용)**: 회계 손익과 프로젝트 손익이 따로 노는 것이 가장 큰 비효율. 두 축의 일원화가 PMS 도입 1차 목표가 되어야 함.

## 발주처/CM/시공사/설계사 별 PMS 적용 차이

| 주체 | 핵심 관심 |
|------|----------|
| 발주처 | 사업 진행률, 예산 집행, 공정 리스크, 일정 보고 |
| CM/CS | 다수 현장 통합 모니터링, 공정·품질·원가 통합 |
| 시공사 | 현장 원가, 외주, 자재, 안전·환경 |
| **설계사무소** | **수주·계약 진행률, 설계 단계별 공수/원가, 외주 협력업체 관리, 도서·도면 버전 관리, 인허가 일정** |

설계사무소는 시공보다 **공수(인력 투입 시간) 기반 원가 관리**의 비중이 크다. 이 부분이 일반 ERP의 제조 원가 모델과 가장 크게 차이 나는 지점.

## PMS 도입 시 주요 기능 요구

### 프로젝트 라이프사이클
- 수주·견적·제안 관리
- 계약·변경계약 관리
- WBS(작업분류체계) 기반 공정 계획
- 진행률(EVM 등) 측정
- 정산·준공 관리
- 사후관리(A/S, 하자보수)

### 비용·자금
- 프로젝트 단위 예산·실적
- 외주 발주·검수·지급
- 직접비/간접비 배부
- 회계 연동 (전표 자동 생성)
- 자금 일정 시뮬레이션

### 인적자원
- 인력 투입(MM, 공수) 관리
- 가용 자원 풀 시각화
- 다중 프로젝트 인력 배분 충돌 점검

### 문서·도면
- 도서/도면 버전 관리, 결재 이력
- 이슈/RFI(설계 질의) 관리
- 외부(발주처·시공사) 공유 권한

### 통합·연동
- 회계 ERP 연동 (필수)
- 그룹웨어·메신저 연동 (요즘 표준)
- BIM/CAD 메타데이터 연계
- 전자세금계산서, 전자결재
- 모바일 (현장 업무)

## 대표 PMIS 솔루션 (참고)
- **휴먼플러스 PMIS** — 건설 PMIS 전문
- **솔브리드플러스 PMS** — 프로젝트 시스템 관리
- **국토교통부 RealPMS** — 공공 정보화 사업 관리, 표준
- 영림원 K-System Ace 의 프로젝트 모듈
- 더존 아마란스10 의 KISS

## 건축설계사무소 관점 — RFP 시 추가 체크리스트

기본 ERP RFP 항목(앞 비교 문서 참고) 외에 PMS 특화 항목:

1. **공수(MM) 기반 원가** 산정 모델 적용 가능 여부
2. **프로젝트 단계별(기획/계획/기본/실시/감리)** 진행률 측정 방식
3. 외주 협력사(구조·설비·전기·조경·인테리어) 공정 종속성 표현 가능 여부
4. 도서/도면 **버전 관리 + 발주처 송부 이력**
5. 인허가 일정·서류 추적 (관청별 워크플로)
6. **부분 인도(分割 정산)** 회계 처리
7. 다중 프로젝트 인력 충돌 시각화
8. 모바일에서 **현장 사진/메모 → 프로젝트 기록** 자동 연결
9. **회계 손익 ↔ 프로젝트 손익 일원화** 기능 (대림 사례 핵심 학습)
10. BIM 모델 메타데이터(객체 단위 수량/물량) 연계 가능성

## 도입 시 흔한 함정
- "현장 데이터 입력 부담"이 도입 후 가장 큰 저항 — UX 단순화 + 모바일 + 자동화로 해결
- 컨설턴트가 자사 표준 프로세스로 강제할 때 설계사 고유 워크플로 손상 가능
- 회계와 프로젝트 손익 일원화가 미흡하면 PMS는 "보고용 화면"으로 전락
- 외주 협력사가 시스템 입력에 협조하지 않아 데이터가 비는 경우 다수 → 협력사 포털을 도입 단계에서 함께 설계해야 함
