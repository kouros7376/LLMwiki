# 바이브 코딩 도구 — 2026

LLM 에이전트가 코드를 직접 쓰고 수정하는 "바이브 코딩(vibe coding)" 도구 6강 체제. 본 위키 운영 환경인 [[Claude_Code]] 가 그중 하나이며, 도구 시장에서도 [[통합형_vs_연동형]] 양극 구도가 그대로 반복됨.

## 시장 점유 (JetBrains 2026.1 조사)

| 도구 | 점유율 | 비고 |
|------|--------|------|
| GitHub Copilot | 29% | 여전히 1위, 엔터프라이즈 표준 |
| Cursor | 18% | 급상승 |
| **Claude Code** | 18% | Cursor 와 동률, 본 위키 환경 |
| Windsurf | 8% | OpenAI 인수 후 통합 진행 |
| 기타 (Antigravity·Kiro 등) | 27% | 신규 진입자 다수 |

## 6강 비교

### [[Claude_Code]] — Anthropic
- 터미널/IDE 확장 기반, **1M 컨텍스트(Opus 4.7)**, MCP 생태계
- 강점: 깊은 추론, 자연어 워크플로, 큰 컨텍스트
- 약점: GUI IDE 약함 (확장으로 보완)
- 가격: Pro $17/월, Max $100~
- 추천: **사내 IT 책임자 자동화 작업, 긴 코드베이스 분석**

### Cursor — Anysphere
- VS Code 포크 기반 풀스택 IDE
- 강점: Composer 모드, Cmd+K 인라인 편집, Tab 완성, 메모리
- 약점: 전용 IDE라 기존 VS Code 환경과 분리
- 가격: Pro $20/월, Pro+ $60/월
- 추천: 웹 개발자, 빠른 프로토타입

### GitHub Copilot — Microsoft
- 다중 모델(GPT-4o/Claude/Gemini), VS Code/JetBrains/Neovim 통합
- 강점: 엔터프라이즈 보안 검증, Copilot Workspace
- 약점: 단순 자동완성 인상, 깊은 추론 약함
- 가격: Pro $10/월, Pro+ $39/월
- 추천: 기존 GitHub 생태계 + 대기업

### Windsurf — Codeium
- 엔터프라이즈 협업 중심
- 강점: 코드베이스 깊은 이해, 다중 파일 프로젝트
- 약점: 점유율 8%
- 추천: 협업 중심 팀, 다중 파일 프로젝트

### Antigravity — Google
- 초심자 친화적 UX
- 강점: 입문 장벽 낮음
- 약점: 복잡한 프로젝트 한계
- 추천: 입문자, 비개발자 시범 도입

### Kiro — AWS
- Spec-Driven Development (명세 기반)
- 강점: 자동 설계 문서, 팀 협업
- 약점: 초기 러닝 커브, 프리뷰 단계
- 추천: 스타트업, 문서화 중시 조직

## 사용자별 권장

| 사용자 유형 | 권장 도구 |
|-----------|----------|
| **사내 IT 책임자가 직접 자동화 도구 만들기** | [[Claude_Code]] (MCP 생태계, 긴 컨텍스트) |
| 프로젝트 빠르게 시작/프로토타입 | Cursor |
| 기존 VS Code/JetBrains 환경 유지 | GitHub Copilot |
| 엔터프라이즈 보안·승인 필요 | Copilot 또는 Windsurf |
| 비개발자 직원도 함께 시작 | Antigravity |
| 명세 → 설계 → 개발 흐름 강제 | Kiro |

## 건축설계사무소 IT 담당자 — 도입 검토

### 자체 개발 vs 외부 도구 분기 — 도구 영역에서도 [[통합형_vs_연동형]] 패턴
- **사내 자동화 스크립트 다수**: [[Claude_Code]] (MCP 로 사내 시스템 직접 연결, 긴 코드베이스)
- **CAD/BIM 매크로 개발**: Copilot + Rhino/Revit 플러그인 → Python/C# 보조
- **반복 행정 업무**: 코드 도구보다 [[워크플로우_자동화_도구]] (n8n 등) 우선 검토

### 점검 사항
1. **데이터 외부 송출 정책** — 도면/회계 코드를 외부 LLM 에 보낼 수 있는가? ([[ERP_RFP_체크리스트]] 의 데이터 외부 반출 항목과 같은 맥락)
2. **로컬 모델 옵션** — 일부 도구는 로컬 LLM 지원
3. **라이선스 모델** — 개인/팀/엔터프라이즈
4. **사내 DB/ERP/PMS 연동** — Claude Code + MCP 가 가장 풍부 (postgres/filesystem/fetch 등)
5. **교육 부담** — 직원 N명에게 어떤 도구 배포할지

### 의외의 연결 — 건축 도메인
[[해외_AEC_사내개발|Foster + Partners Applied R+D 팀]] 도 사내 자동화 도구를 적극 만드는데, **외부 의존을 Rhino + 상용 BIM 두 가지로만 한정**. 같은 분기 기준이 도구 시장에도 적용됨 — "기성 도구로 충분한 영역 vs 사내 맞춤 필요한 영역" 분리.
