# ERP / PMS RFP 체크리스트

[[더존_아마란스10]], [[영림원_K-System_Ace]] 등 어느 벤더든 RFP 단계에서 공통으로 검증해야 할 항목 모음. 누락 시 도입 후 1~2년차에 대형 갈등 발생.

## 일반 ERP 항목

1. 프로젝트(현장) 단위 매출/원가 마감 자동화
2. 결재 모바일 지원 + 외근/재택에서 작동
3. 외주 비용 → 계약 → 세금계산서 → 지급까지 자동 연결
4. **SSO / Active Directory 연동**
5. 메신저/메일/문서관리 통합 또는 표준 API 제공
6. 데이터 외부 반출(Export) 정책 및 형식
7. 마이그레이션 시 기존 회계/PMS 데이터 이관 범위와 책임
8. 패치/업그레이드 주기 및 다운타임 정책
9. 보안 인증 (CSAP, ISO27001, ISMS 등)
10. SLA 및 장애 보상 조건

## PMS 특화 항목 (건축설계사무소)

→ 자세한 배경은 [[건설_PMS]]

11. **공수(MM) 기반 원가** 산정 가능 여부
12. **설계 단계별** (기획/계획/기본/실시/감리) 진행률 측정
13. 외주 협력사(구조·설비·전기·조경·인테리어) 공정 종속성
14. **도서/도면 버전 관리 + 발주처 송부 이력**
15. 인허가 일정·서류 추적 (관청별 워크플로)
16. 부분 인도(분할 정산) 회계 처리
17. 다중 프로젝트 인력 충돌 시각화
18. **모바일 현장 사진/메모 → 프로젝트 기록** 자동 연결
19. **회계 손익 ↔ 프로젝트 손익 일원화** ← [[정보_일원화_패턴]] 핵심
20. BIM 모델 메타데이터 연계

## 운영·문화 항목 (자주 빠지는 영역)

21. 외주 협력사 포털/입력 환경 — 미입력 시 데이터 빔 ([[건설_PMS]] 함정)
22. 도입 후 1~2년 안정화 운영 책임 분담 (벤더 vs 자사 IT)
23. 컨설턴트의 표준 프로세스 강제 vs 설계사 고유 워크플로 보존
24. 옛 시스템 종료 일정 — 일원화 후에도 못 끄면 이중 운영 ([[정보_일원화_패턴]] 함정 #3)
25. 사용자 교육 시간·범위·재교육 주기

## 벤더별 추가 확인 (양극 구도)

→ [[통합형_vs_연동형]]

### [[더존_아마란스10]] (통합형) RFP 추가
- KISS 모듈의 건축 PMS 적합성 — 데모 시 본인 사례 그대로 입력 요청
- ONE AI 의 회계 데이터 분석 정확도 — 본인 데이터로 시연 요구
- 락인(Vendor Lock-in) 시 데이터 반출 가능 형식

### [[영림원_K-System_Ace]] (연동형) RFP 추가
- K-System GW (그룹웨어) 도입 비용 별도 산정
- 외부 시스템(BIM/CAD) 연동 사례 — 동종 업계 사례 요청
- 컨설팅 역량 — PM 이력서 사전 검토

## 데모 시 반드시 본인 데이터로 시연 요청

벤더 데모는 보통 자사 표준 케이스로 잘 보임. RFP 단계에서 다음을 본인 회사 실제 데이터로 시연 요구:
- 실제 프로젝트 1개의 수주 → 정산 끝까지
- 외주 비용 청구 → 지급 흐름
- 월 마감 시 회계·프로젝트 손익 일치 확인 화면

벤더가 "그건 추가 컨설팅" 이라고 답하면 RFP 단계에서 견적에 포함시킬 것.
