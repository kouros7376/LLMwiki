# 정림건축 혁신본부 + JAdU

국내 건축설계사무소 중 가장 가시적인 사내 R&D + 교육 프로그램 운영. **Junglim Innovation 본부** 가 사내 개발, **JAdU(정림 아카데미 디지털 유닛)** 가 외부 인재 양성. 본 위키 [[정보_일원화_패턴]] 의 BIM 도메인 + [[지식의_복리]] 의 조직 적용을 한국 사례로 보여줌.

## Junglim Innovation 본부

**공식 블로그**: <https://innovation.junglim.com>

### 대표 사내 개발 사례 — BIM 규모검토 Package

**문제**: 초기 설계 단계에서 CAD/SketchUp/Rhino/Excel/PowerPoint 다중 도구 사용 → 면적 변경 시 수치 불일치, 반복 작업

**해법**: Revit 2017 기반 자체 개발 패키지, **단일 정보원(Single Source of Truth)** 원칙
→ 본 위키 [[정보_일원화_패턴]] 의 BIM 도메인 구현 — [[건설_PMS|대림산업의 손익 일원화]] 와 같은 본질의 다른 도메인.

**핵심 기능**:
| 기능 | 효과 |
|------|------|
| 매스↔평면↔면적 연동 | 매스 수정 시 면적 자동 반영 |
| 태그↔용도 연동 | 변경 시 별도 수정 불필요 |
| Material 기반 컬러링 | 뷰별 작업 제거 |
| Excel 연계 | 초기 설정 후 데이터만 갱신 |
| **BIM Law 데이터** 불러오기 | **법규 자동 체크** |

**조직**: Junglim Innovation 본부 BIM팀 (BIM Manager 정혜수)

**확산**: J-edu 교육 영상으로 업계 배포 — [[지식의_복리]] 의 "사내 자료 → 외부 공유로 누적" 패턴

### 사용 기술 스택
- **Revit + Dynamo** (메인)
- Python (Dynamo 노드, Revit API)
- C# (Revit API)
- D5 렌더링 + AI

→ [[해외_AEC_사내개발|Foster + Partners]] 의 클라우드/GPU/분산 컴퓨팅 인프라와 비교하면 **Revit 중심 통합형 + 일부 외부**. 양 사례의 [[통합형_vs_연동형]] 위치가 다름.

## JAdU (정림 아카데미 디지털 유닛)

**홈페이지**: <https://www.jacademyhub.com/>

### 목표
디지털 트윈, BIM, 컴퓨테이셔널 디자인 등 **하이테크 친환경 설계 인재 양성**.

### 운영 현황
- 2024년 5월 1기 → 2025년 3기 → **2026년 4기 모집 중**
- 국가 인증: K 디지털 트레이닝 선도기업 아카데미 / 첨단산업·디지털 선도기업 아카데미

### 커리큘럼
**600시간 집중 + Project Based Learning**
- 모듈러 디지털 트윈 프로젝트
- ZEB 친환경 건축 시뮬레이션
- BIM 교육 + 자격증
- AI 특강 + D5 렌더링
- 3D 프린팅 실습

### 취업 연계
JAdU 수료생 → 정림건축 입사 사례 다수. **JAH(정림아카데미허브)** 가 교육 프로그램 기획·운영.

### JAdU 의 메타 의미 — [[지식의_복리]] 조직 적용
정림 사내 60여 년의 노하우를 600시간 교육으로 압축 → 외부 인재가 그 위에서 출발 → 일부가 정림 입사 → 정림은 첫날부터 "이미 학습된 인력" 확보.

→ 이건 [[지식의_복리]] 의 조직·인적자원 차원 구현. 본 위키 [[CLAUDE]] 가 Claude 에게 운영 규칙을 압축하는 것과 같은 패턴.

## 정림 + 티랩스 협약
- 메타버스 서비스 개발 + 인프라 구축 사업 협력
- 디지털 트윈 영역 강화

## 한국 시장 위치
- 정림건축 + 간삼건축 = **국내 BIM 적용 실적 1위 (공동)**
- 10인 미만 소규모 사무소 59% / BIM 실적 중견 이상 대부분 → **디지털 격차 큼**

→ [[ERP_벤더_비교]] 의 도입 규모 분기와 같은 패턴 — 큰 회사는 [[해외_AEC_사내개발|Foster 모델]] 처럼 사내 개발, 작은 회사는 외부 도구 의존.

## 다른 한국 건축사 — 정보 갭
- 희림, 삼우, 해안건축 등은 사내 개발 도구·블로그 공개 거의 없음
- 추적 채널: 빌딩스마트협회 The BIM 잡지, 대한건축사협회 건축사신문, 한국BIM학회 발표

## 본 위키 컨텍스트와의 연결

### 정림 BIM 규모검토 Package = [[정보_일원화_패턴]] BIM 도메인
- 다중 도구 분산 입력 → 단일 정보원 = ERP 의 손익 일원화와 본질 동일
- 한 번 정리 후 변경은 한 곳만 → 다른 모든 곳 자동 반영

### J-edu 모델 = [[지식의_복리]] 조직 적용
- 사내 개발 → 교육 영상 공개 → 업계 도입
- 본 [[CLAUDE|kouwiki 운영 규칙]] 자체가 같은 패턴 (사내 자료 → Claude 가 학습 → 외부 사용자 기여 흡수)

### 정림 Revit 통합형 vs Foster Hermes 연동형 = [[통합형_vs_연동형]]
- 정림: Revit 한 도구에 패키지 형태로 통합 (벤더 락인 일부 수용)
- Foster: 외부는 Rhino + BIM 만, 나머지는 사내 클라우드 인프라 (선별적 하이브리드)

→ 300명 규모 사무소의 1차 분기 결정 사항. 자세한 가이드는 [[건축IT_사내개발_전략]].

## 300명 규모 사무소가 정림 모델에서 차용할 5가지

1. **소규모 디지털 디자인 그룹 1~3명** 부터 시작 (정림 혁신본부 BIM팀 = 1인 시작)
2. **첫 사내 도구는 [[정보_일원화_패턴]] 위반 영역** 부터 (반복 입력, 수치 불일치)
3. **사내 교육 프로그램** 으로 도구 사용 확산 (J-edu 모델)
4. **사내 개발 → 블로그 공개** 로 채용 브랜딩 (정림이 검증)
5. **첫 도구는 Revit + Dynamo + Python** (가장 ROI 빠름)
