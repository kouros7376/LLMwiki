# 워크플로우 자동화 도구

전산실 반복업무 자동화 도구 비교. **n8n / Power Automate / Make / Zapier / Retool** 5종이 실무 후보. 데이터 정책과 사내 환경에 따라 선택. 시나리오는 [[자동화_시나리오]] 참고. **[[RPA]]와의 분기**: API 있고 데이터 흐름 핵심이면 본 페이지 도구, GUI만 있고 사람 클릭 필수면 [[RPA]] (정부 사이트 한정). 둘 다 가능하면 **본 페이지 도구 우선** (유지비·내구성 압도적).

## 5종 비교

### n8n (오픈소스)
- **포지션**: 오픈소스(Fair-code), 셀프호스트, 노코드+로우코드 혼합
- **강점**:
  - **셀프호스트** → 데이터 외부 유출 없음 ([[ERP_RFP_체크리스트]] 의 데이터 외부 반출 정책 충족)
  - 400+ 통합 노드 (Slack, Notion, Postgres, OpenAI, Google Workspace 등)
  - 커스텀 JS/Python 노드 → **레거시 ERP·내부 DB 연동** 자유
  - 로컬 LLM(Ollama) 연결 가능
- **약점**: 셀프호스트 운영 부담 (서버, 백업, 모니터링)
- **가격**: 셀프호스트 무료, 클라우드 $24/월~
- **추천**: **건축설계사무소 IT 담당자에게 1순위** (사내 데이터 정책 엄격 + 사내 시스템 풍부)

### Microsoft Power Automate
- **포지션**: MS 365 통합형 RPA + 워크플로우
- **강점**:
  - **Office 365 / Dynamics 365 / SharePoint** 깊은 통합
  - **Desktop Flow(RPA)** — UI 자동화로 레거시 시스템 조작 (구식 ERP 대응)
  - **AI Builder** — 송장·계약서 등 문서에서 데이터 추출
  - Copilot 통합 (자연어 플로우 생성)
- **약점**: MS 외 시스템 connector 추가 비용, 라이선스 복잡
- **가격**: 사용자당 $15/월 (Premium), Process plan $150/월~
- **추천**: MS 365 위주 + RPA 필요 시

### Zapier
- **포지션**: SaaS 통합 표준, 5,000+ 앱
- **강점**: 가장 많은 앱, 직관적, 비개발자 친화
- **약점**: 비싼 편(작업당 과금), 복잡한 로직 어려움, 셀프호스트 불가
- **가격**: $19.99/월~, 트랜잭션 기반
- **추천**: SaaS 위주 환경, 빠른 시작, 비개발자 직원 자동화

### Make (구 Integromat)
- **포지션**: 시각적 시나리오, Zapier 대안
- **강점**: 시각적 UI, Zapier 대비 저렴, 분기·반복 표현 쉬움
- **약점**: 셀프호스트 불가, 한국어 자료 적음
- **가격**: 무료 1,000 ops/월~, $10.59/월~
- **추천**: Zapier 대비 비용·UI 선호 시

### Retool
- **포지션**: 사내 도구 빠른 구축 (워크플로우 + admin UI)
- **강점**: DB 직접 연결 → 빠른 사내 admin 화면, 워크플로우 + UI 통합
- **약점**: 워크플로우 단독 비교에서는 n8n/Make 우위
- **가격**: 무료 5명까지, $10/사용자~
- **추천**: 사내 데이터 조회/수정 UI 가 함께 필요할 때

## 선택 매트릭스

| 조건 | 우선 도구 |
|------|----------|
| 데이터 외부 송출 금지 | **n8n (셀프호스트)** |
| MS 365 중심 환경 + RPA | Power Automate |
| 비개발자 직원도 만들고 싶음 | Zapier / Make |
| 사내 admin UI 함께 필요 | Retool |
| AI(LLM) 워크플로우 깊이 활용 | n8n + 외부 LLM API |

## 본 위키 컨텍스트와의 연결

### [[통합형_vs_연동형]] 패턴 — 워크플로우 도구에서도 반복
- **통합형**: Power Automate (MS 365 묶음 안에서 모든 것)
- **연동형**: n8n + Claude API + Notion API 조립

→ ERP 시장의 [[더존_아마란스10|더존]] vs [[영림원_K-System_Ace|영림원]] 분기와 정확히 같은 형태.

### [[정보_일원화_패턴]] 적용
워크플로우 자동화의 본질은 "흩어진 시스템 데이터를 한 흐름으로 묶는 것" = [[정보_일원화_패턴]] 의 자동화 도메인 구현.

### [[Claude_Code]] 와의 분기
- 코드 작성 자체 → [[Claude_Code]]
- 트리거·스케줄·여러 시스템 연결 → 워크플로우 도구
- **연결**: n8n 안에서 OpenAI/Anthropic 노드 호출 가능, 또는 Claude Code 가 작성한 스크립트를 n8n 이 스케줄링

## 흔한 함정

1. **자동화 자체가 목적이 됨** — "월 100개 작업 정도의 간단한 자동화로 시작" 권장
2. **에러 처리 부재** — 한 번 깨지면 누구도 모름 → 모니터링·알림 필수
3. **권한·시크릿 관리 허술** — n8n credentials 백업 분리 필수
4. **셀프호스트 운영 인력 미배정** — n8n 도입 시 운영 SRE 시간 산정 필요

→ [[건강검진]] 의 도메인 적용: 워크플로우 자동화도 깨진 링크/고립 노드와 같은 점검이 필요. n8n 의 실패한 실행 / 미실행 워크플로 정기 점검.
