# Graphify: 멀티모달 코퍼스를 질의 가능한 지식 그래프로 변환하는 AI 코딩 어시스턴트

Graphify는 단순한 README 생성기를 넘어, 코드, 문서, 논문, 이미지, 영상, 오디오 등 다양한 형태의 데이터를 하나의 통합된 그래프 메모리로 엮어 AI 코딩 어시스턴트가 구조 중심으로 탐색하도록 돕는 **graph-first orchestration layer** 입니다.

## 핵심 기능

### 1) 멀티모달 Ingestion: "파일 업로드"가 아닌 "추출 경로 선택"

Graphify는 다양한 파일 유형을 처리할 뿐만 아니라, 각 유형별 처리 비용 구조를 명확히 분리합니다. 코드는 AST 추출로 로컬 처리하고, 문서/이미지 등은 의미 추출 단계로 넘어가며, 영상/음성은 Whisper 전사 후 의미 추출에 연결됩니다. 이를 통해 가장 저렴한 해석 경로를 먼저 택하고, 비싼 의미 추출은 필요한 모달리티에만 적용하는 하이브리드 인덱서로 작동합니다.

### 2) 코드 추출: 단순 심볼 인덱싱을 넘어 "존재 이유"까지 포착

`extract.py` 모듈은 언어별 추출기와 generic extractor를 함께 사용하며, `# NOTE`, `# IMPORTANT`와 같은 rationale comment를 별도 신호로 취급하여 "왜 그렇게 작성되었는가"를 그래프에 포함시키려 합니다. 또한 cross-file import resolution을 통해 구조적 관계를 살아있는 그래프로 만듭니다.

### 3) Edge Taxonomy: EXTRACTED, INFERRED, AMBIGUOUS로 신뢰도 명시

Graphify는 확신의 출처를 명확히 합니다. `EXTRACTED`는 소스에서 직접 확인된 사실, `INFERRED`는 합리적 추론, `AMBIGUOUS`는 검토가 필요한 관계를 나타냅니다. `INFERRED`에는 confidence score가 붙으며, 클러스터링은 임베딩이 아닌 그래프 토폴로지를 기준으로 합니다.

### 4) Assistant Behavior Modification: 그래프 우선 탐색 습관 형성

Graphify는 단순히 아티팩트를 생성하는 것을 넘어, 플랫폼별 hook/rule/skill 설치를 통해 AI 어시스턴트가 raw grep/search 전에 `GRAPH_REPORT.md`를 먼저 보도록 흐름을 재배선합니다. 이는 에이전트의 탐색 순서를 graph-first로 바꾸는 핵심 차별점입니다.

### 5) 넓은 출력 Surface: Report, JSON, HTML, SVG, GraphML 등 다양한 형식 지원

기본 산출물은 `GRAPH_REPORT.md`, `graph.json`, `graph.html`이며, `export.py`를 통해 SVG, GraphML, Neo4j, Obsidian, Wiki, Canvas 등 다양한 형식으로 내보낼 수 있습니다. 이는 한 번 인덱싱한 그래프를 광범위하게 재사용할 수 있게 합니다.

### 6) 운영 레이어: `watch.py`와 `serve.py`를 통한 지속형 Graph Substrate

`watch.py`는 코드 변경 시 AST-only rebuild를 즉시 수행하고, 문서/이미지 변경 시 LLM 재추출 필요성을 알립니다. `serve.py`는 `graph.json`을 MCP 서버로 노출하여 에이전트가 구조화된 질의를 수행할 수 있게 합니다. 이를 통해 Graphify는 일회성 분석 툴이 아닌, 변경을 따라가고 질의를 받는 지속형 graph substrate가 됩니다.

### 7) 벤치마크 메시지의 정직함

`worked/karpathy-repos/review.md`에서 제시된 평균 질의당 71.5x 토큰 절감 효과는 과장되지 않았습니다. 첫 추출 비용은 있지만, 그래프 생성 후 후속 질문에서 절감 효과가 커집니다. 작은 코퍼스에서는 압축보다 구조적 이해가 더 큰 가치일 수 있습니다.

### 8) 보안/프라이버시 고려

`security.py`와 `detect.py`는 secret 파일 스킵, IP/메타데이터 엔드포인트 차단 등 기본적인 방어선을 갖춥니다. 다만, 문서/이미지 semantic extraction은 호스트 어시스턴트 제공업체의 모델 API를 사용하므로, PHI/임상 자료 등 민감 정보 처리 시에는 주의가 필요합니다.

## 기술 스택

Python `>=3.10`을 기반으로 NetworkX, tree-sitter, `graphifyy` 패키지 등을 활용합니다. 다양한 언어 문법을 지원하며, 필요에 따라 Leiden, pypdf, faster-whisper 등 추가 기능을 설치할 수 있습니다.

## 아키텍처

`detect → extract → build → cluster → analyze → export` 파이프라인을 따릅니다. Detection layer에서 코퍼스 정제를 먼저 수행하고, Extraction layer에서 코드와 문서의 핵심 가치를 추출합니다. Build/cluster/analyze 중간층은 비교적 단순하며, Export layer에서 다양한 형식으로 결과물을 내보냅니다.

## 사용법 및 설정

`pip install graphifyy`로 설치 후, `graphify install`로 플랫폼별 skill/rule을 설치합니다. `graphify .` 명령으로 그래프를 생성하고, `graphify query`, `path`, `explain` 등으로 질의할 수 있습니다. `--watch` 옵션으로 변경 사항을 감지하거나, `python -m graphify.serve`로 MCP 서버를 실행할 수 있습니다.

## 개인적 통찰

**의료 AI 관점:** 구조적 사실과 의미 추론을 분리하고 audit trail을 남길 수 있어 매력적이나, semantic extraction의 외부 API 의존성으로 인해 컴플라이언스 검토가 필요합니다.

**Bioinformatics 관점:** 논문-코드 연결 자동화에 유용하나, R 미지원 및 workflow DSL 미흡은 한계입니다.

**Autonomous Agent 관점:** artifact 생성 넘어 agent 행동 자체를 바꾸고, 구조화된 질의를 지원하며, 지속형 메모리 레이어로 기능하는 점에서 가장 강력한 잠재력을 보입니다.

## 최종 평가

Graphify는 코드, 문서, 논문 등이 혼합된 멀티모달 코퍼스와 AI 코딩 어시스턴트의 graph-native 탐색 워크플로에 특히 강력한 도구입니다. 단순한 오픈소스 레포 설명기를 넘어, AI 코딩 어시스턴트를 위한 인덱싱 및 오케스트레이션 계층으로 보는 것이 가장 정확합니다.
