[[해외_AEC_사내개발|Foster + Partners]] 모델과 [[정림건축_혁신본부|정림 모델]], [[AEC_자동화_플랫폼|외부 플랫폼]] 을 종합한 **300명 규모 건축설계사무소 전산실 책임자용 사내개발 로드맵**. 본 위키 메타 페이지([[정보_일원화_패턴]], [[통합형_vs_연동형]], [[지식의_복리]]) 의 AEC 도메인 종합 적용.

## 1차 분기 — 사내 개발 vs 외부 도구

### Foster 의 4가지 분기 기준 (그대로 적용 가능)
어떤 영역을 사내 개발할지는 다음 4가지로 판단:
1. **최종 사용자(설계팀) 중심 맞춤** 필요한가?
2. **상호운용성** 필요한가? (여러 도구 간 데이터 교환)
3. **클라우드 인프라 통합** 필요한가?
4. **장기 피드백 순환** 필요한가? (설계팀과 지속 협업)

→ 4가지 모두 YES → 사내 개발. 1~2개만 YES → 외부 도구.

## 단계적 로드맵 — 3단계

### 1단계 (0~6개월): 즉시 시작 가능한 가성비 영역
**목표**: 채용·개발 인프라 없이 효과 즉시 입증.

1. **n8n 셀프호스트 1대 구축** + [[자동화_시나리오|시나리오 2(일일 보고서)]] PoC
   - 비용: 서버 1대, IT 담당자 시간 80h
   - 효과: 매일 30분 절감, 정보 가시성 즉시 향상
2. **[[Claude_Code]] + MCP 도입** (전산실 책임자 직접 사용)
   - 비용: 라이선스 월 17~100$
   - 효과: 자동화 스크립트, 사내 도구 프로토타입 작성 속도 ↑
3. **Speckle PoC** ([[AEC_자동화_플랫폼]])
   - 비용: 오픈소스, 셀프호스트 가능
   - 효과: BIM 도구 간 데이터 흐름 즉시 개선, 락인 회피 검증
4. **[[RPA|선택적 RPA PoC]]** — 정부 사이트 1건 한정 (4대보험 EDI 또는 홈택스)
   - 비용: Power Automate Desktop 무료 또는 AutomateOne/CheckMATE PoC 500-1,500만원
   - 효과: API 부재 영역의 자동화 입증. 본 사무소 [[RPA|3-tier 아키텍처]] 의 RPA 자리 검증
   - 주의: 봇 개수 X, **"절감 시간 + 오류율 감소"가 KPI** (Gartner Agent-washing 경고)

### 2단계 (6~18개월): 디지털 디자인 그룹 신설
**목표**: 1~3명 전담 인력으로 사내 개발 본격화.

1. **사내 디지털 디자인 그룹 1명 채용 또는 보직 전환** (정림 혁신본부 BIM팀 = 1인 시작)
   - 첫 인력은 **건축 백그라운드 + Revit/Dynamo 능력자** 우선 (외부에서 데려오기 어려움)
   - 또는 사내 BIM 잘하는 직원 보직 전환
2. **첫 사내 도구 = [[정보_일원화_패턴]] 위반 영역**
   - 정림처럼 BIM 규모검토 Package 같은 단일정보원 도구
   - 또는 사무소 고유 결재·도서 흐름 자동화
3. **블로그 운영** — 정림 J-edu 모델 차용 (사내 도구 → 외부 공개)
   - 채용 브랜딩 효과 (→ 정림이 검증)
4. **사내 교육 프로그램 시작** — 직원 대상 BIM 자동화 커리큘럼

### 3단계 (18~36개월): 사내 개발 + 외부 통합
**목표**: 사내 자산화 + 외부 플랫폼 활용 균형.

1. **사내 도구 카탈로그 + 운영 가이드 표준화** ([[건강검진]] 자동화)
2. **외부 도구와의 통합 깊이** (Speckle 위에서 사내 도구 운영)
3. **AI 기반 자동화 본격 — [[자동화_시나리오|시나리오 5(회의록 ingest)]] 등**
4. **JAdU 같은 외부 교육 프로그램 검토** (인재 양성으로 채용 풀 확장)

## 채용·인재 전략

### 1~3명 채용 시 우선순위
1순위: **건축 백그라운드 + Revit/Dynamo + 약간의 Python** (사무소 워크플로 이해 + 자동화 가능)
2순위: 컴퓨테이셔널 디자인 전공 신입 (JAdU 출신 등)
3순위: 일반 개발자 + 건축 학습 (가장 어려움)

→ 1순위가 가장 적합하지만 시장에서 가장 희소. 정림 JAdU 출신을 노리는 것이 현실적.

### 외부 채용 어려울 시 — 보직 전환
- 사내 BIM 잘하는 직원 1명 → 디지털 디자인 그룹으로 보직 전환
- [[Claude_Code]] + MCP 로 코드 작성 부담을 LLM 이 보조 → 비개발자도 도구 개발 가능

## 본 위키 메타 페이지 종합 적용

### [[정보_일원화_패턴]] — AEC 사내 개발의 1순위 영역 식별
사내 도구 개발 시 첫 후보는 항상 **"같은 사실을 두 번 입력하는 영역"**. 정림 BIM Package 도, ERP 손익 일원화도, 본 위키 [[CLAUDE]] 도 모두 같은 패턴. 이 영역이 가성비 최고.

### [[통합형_vs_연동형]] — 1차 분기 결정
사무소가 통합형(Autodesk Construction Cloud + Revit 묶음)으로 갈지, 연동형(Speckle + 사내 도구 + 외부 플랫폼 조립)으로 갈지가 1차 분기. **연동형이 향후 [[AEC_자동화_플랫폼|NXT BLD 트렌드]] 와 일치** — Applications on Demand 흐름.

### [[지식의_복리]] — 사내 개발의 장기 가치
사내 개발은 1년 안에는 ROI 안 나옴. 그러나 [[해외_AEC_사내개발|Foster Hydra]] 처럼 5년 누적되면 **130만 시뮬레이션 같은 자산**이 됨. 단기 비교는 무의미, 장기 누적이 본질.

### [[건강검진]] — 사내 도구 무결성
사내 도구도 [[건강검진]] 필요:
- 안 쓰는 도구 정리
- 깨진 워크플로 알림
- 도구 카탈로그·문서 정기 갱신

## 의외의 연결 — 본 사무소 ERP 마이그레이션과의 관계

[[ERP_벤더_비교|ERP 의사결정]] 과 사내개발 전략은 **같은 1차 분기**:
- 통합형 ERP([[더존_아마란스10]]) + 통합형 BIM(Autodesk Construction Cloud) → 한 벤더 두 묶음, 락인 강화
- 연동형 ERP([[영림원_K-System_Ace]]) + 연동형 AEC(Speckle + 사내) → 사내 IT 역량 핵심

→ 두 결정을 동시에 하면 **사무소 전체 IT 전략의 일관성** 확보 가능. ERP 통합형 + AEC 연동형은 내부 모순 (한쪽은 락인 수용, 한쪽은 회피).
