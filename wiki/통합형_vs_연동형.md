# 통합형 vs 연동형

소프트웨어 플랫폼 시장에서 반복되는 양극 구도. ERP 시장의 [[더존_아마란스10]] vs [[영림원_K-System_Ace]] 구도가, AI 도구 시장의 LLM Wiki vs MCP 생태계 구도와 **같은 형태로 반복** 된다는 본 위키의 핵심 통찰.

## 두 축 정의

### 통합형 (Integrated, "All-in-One")
한 플랫폼이 ERP·그룹웨어·문서관리·결재·메신저·AI 까지 자체적으로 다 제공.

- **ERP**: [[더존_아마란스10]] — ERP+그룹웨어+문서+ONE AI 단일 플랫폼
- **AI/지식**: 노션 AI, ChatGPT Workspace, MS Copilot 같은 단일 SaaS 묶음
- **강점**: 단일 로그인, 일관된 UX, 데이터 중복 제거([[정보_일원화_패턴]] 의 자연스러운 구현)
- **약점**: 한 영역(예: 산업특화 PMS)에서 깊이 부족, 락인 위험, 변화 속도 벤더 의존

### 연동형 (Composable, "Best-of-Breed")
좋은 단일 기능 도구를 표준 인터페이스로 연결.

- **ERP**: [[영림원_K-System_Ace]] + 외부 그룹웨어/메신저 + BIM/CAD 연동
- **AI/지식**: Claude + MCP 생태계(filesystem, fetch, postgres, puppeteer, context7, memory) + Obsidian 보관함 ← **본 kouwiki 가 이 구조**
- **강점**: 영역별 최고 도구 채택, 산업특화 깊이, 락인 회피, 변화 빠른 영역 따라가기 쉬움
- **약점**: 통합 책임이 사용자/통합자에게, SSO·데이터 정합성 비용

## 의외의 동형성 — ERP ↔ AI 도구

| 측면 | ERP 시장 | AI 도구 시장 |
|------|----------|-------------|
| 통합형의 표상 | [[더존_아마란스10]] (ONE AI 포함) | 노션 AI, ChatGPT Workspace |
| 연동형의 표상 | [[영림원_K-System_Ace]] + 외부 도구 | [[LLM_Wiki_개념]] + Claude + MCP |
| 회의록 자동화 (예시 기능) | ONE AI 회의록 자동작성 (벤더 내장) | Whisper + Claude + Obsidian (사용자 조립) |
| 결재/문서 자동검증 | ONE AI 결재 자동검증 (벤더 내장) | Claude + MCP postgres + 사용자 룰 |
| ERP 데이터 분석 | ONE AI ERP 데이터 분석 (벤더 내장) | Claude + MCP postgres |

**결론**: 기능 자체는 양쪽 모두 가능. 차이는 **누가 책임지고 누가 진화시키는가**.

## 의사결정 가이드 (전산실 총괄 관점)

| 조건 | 권장 |
|------|------|
| 인력·역량 한정, 통합 책임 외주화 선호 | **통합형** |
| 산업특화 깊이 + 외부 도구 다양성(BIM/CAD/PMS) 필요 | **연동형** |
| 변화 빠른 영역(AI 도구 신구) 따라가기 원함 | **연동형** |
| 보안·감사·인증을 단일 벤더로 책임지길 원함 | **통합형** |
| 5~10년 락인 부담을 받아들임 | **통합형** |
| Best-of-breed 자유도가 핵심 | **연동형** |
| 직원 IT 리터러시 낮음 → 일관 UX 중요 | **통합형** |
| 사내 IT/전산실 역량 강함 | **연동형** |

## 1차 분기점

ERP 의사결정 시 단순히 "더존이냐 영림원이냐"가 아니라 **"통합형으로 갈 것인가, 연동형으로 갈 것인가"** 가 1차 분기점. 이 분기를 먼저 결정해야 비교 항목([[ERP_벤더_비교]], [[ERP_RFP_체크리스트]])이 의미 있어짐.

```
                  [ ERP 의사결정 ]
                         │
        ┌────────────────┴────────────────┐
        │                                  │
   통합형 선호                          연동형 선호
        │                                  │
   [[더존_아마란스10]]                  [[영림원_K-System_Ace]]
        │                                  │
   추가 검토:                          추가 검토:
   - PMS 깊이 보완                     - 그룹웨어/메신저 외부 도구
   - 락인 시 데이터 반출                - SSO·정합성 자동화
```

## 본 kouwiki 가 연동형인 이유

본 위키 자체가 연동형의 살아있는 사례:
- **knowledge layer**: Obsidian 보관함 (마크다운 + `[[wikilinks]]`)
- **AI 인터페이스**: Claude Code (VSCode 확장)
- **외부 연결**: MCP 6개 — fetch / filesystem / puppeteer / postgres / context7 / memory
- **운영 규칙**: [[CLAUDE]] (Schema)

같은 가치를 통합형으로 풀려면? → 노션 AI 한 곳에 다 두거나, [[더존_아마란스10]] ONE AI 같은 사내 솔루션 사용. 락인 vs 자유도 분기.

## 하이브리드 가능성

순수 통합형/연동형은 양 극단. 실제 도입은 보통 하이브리드:
- 통합형 ERP 도입 + 외부 BIM/CAD 도구 연동 (락인 일부 받아들이고 일부 자유)
- 연동형 ERP 도입 + 같은 벤더의 그룹웨어 패키지로 묶기 (자유도 일부 포기)

→ 분기 후 각 영역에서 **얼마나 깊게 통합/연동할지** 가 2차 결정.
